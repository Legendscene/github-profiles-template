## 🔧 Complete Customization Guide

### Step 1: Basic Information
| What to Replace | Where It Appears | Replace With | Example |
|----------------|-----------------|--------------|---------|
| `<your-name>` | Table header heading | Your full name | Alex Rivera |
| `yourusername` | ALL URLs, stats, avatar, trophies | Your GitHub username | alexrivera |
| `<your-tagline>` | Italic text under name | Your title/description | Building the future with React |
| `your@email.com` | (Add your own email badge) | Your email | alex@example.com |
| `yourblog.com` | Blog post section URLs | Your blog URL | https://alexrivera.dev |

### Step 2: Social Links
Replace these in the badge section below the avatar:
- `https://twitter.com/yourusername` → `https://twitter.com/YOUR_HANDLE`
- `https://linkedin.com/in/yourusername` → `https://linkedin.com/in/YOUR_LINKEDIN`
- `https://instagram.com/yourusername` → `https://instagram.com/YOUR_HANDLE`
- `https://youtube.com/@yourusername` → `https://youtube.com/@YOUR_CHANNEL`

### Step 3: Tech Stack Badges
Replace the example badges with your actual tech stack:
- Each badge follows this pattern: `![Label](https://img.shields.io/badge/Category-Name-HEXCODE?style=for-the-badge&logo=logo-name&logoColor=white)`
- Change the badge color (the hex code after the name) to match your brand
- Find available logos at `https://simpleicons.org/`
- Remove badges you don't use, add ones you do

### Step 4: GitHub Stats & Trophies
Replace `yourusername` in all three stats/trophy URLs:
- `https://github-readme-stats.vercel.app/api?username=yourusername&...`
- `https://github-readme-stats.vercel.app/api/top-langs/?username=yourusername&...`
- `https://github-profile-trophy.vercel.app/?username=yourusername&...`

The `column=7` parameter in trophies fits 7 trophies in one row. Adjust to `column=4` or `column=5` on smaller screens.

### Step 5: Blog Posts Section
For auto-updating blog posts:
1. Set up the **Blog Post Workflow** from `https://github.com/marketplace/actions/blog-post-workflow`
2. Add a `.github/workflows/blog-posts.yml` file
3. The `<!-- BLOG-POST-LIST:START -->` and `<!-- BLOG-POST-LIST:END -->` comments act as markers — the action fills content between them
4. Without the action, manually replace the placeholder posts with your own links

### Step 6: Header & Footer Text
- Header: Change `%3C%2F%3E` (URL-encoded `</>`) to your own symbol or text
- Footer: Change `Thanks%20for%20visiting!` to any message
- Use `https://www.urlencoder.org/` to encode your text

## 💡 Pro Tips

1. **Gradient colors**: The gradient is auto-generated from Capsule Render. To use custom gradient colors instead, replace `color=gradient` with `color=0:FF6B6B,100:4ECDC4` (two hex colors).
2. **Avatar shape**: The `border-radius:50%` makes the avatar circular. Change to `border-radius:10%` for rounded square, or remove it for square.
3. **Blog workflow**: The BLOG-POST-LIST markers work with GitHub Actions. Without the workflow, just delete the markers and keep your static links.
4. **Add more socials**: Duplicate the `<a>` tag pattern to add Discord, TikTok, Dev.to, or any platform. Flat-style badges (`style=flat`) keep it clean.
5. **Mobile responsive**: The table layout may not look great on mobile. Consider testing how it renders or replacing the `<table>` with a `<div>`-based layout.

## 🚀 One-Click Deploy

1. Copy everything inside the **"Ready-to-Use Code"** block above
2. Go to your profile repo: `github.com/YOUR_USERNAME/YOUR_USERNAME`
3. Open `README.md` and paste over everything
4. Press **Ctrl+H** (or **Cmd+H** on Mac) → find `yourusername` → **Replace All** with your actual GitHub username
5. Replace `<your-name>`, `<your-tagline>`, and blog post URLs
6. Customize the tech stack badges to match your actual skills
7. Commit — your new profile is LIVE! 🎉

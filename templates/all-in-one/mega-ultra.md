### Step 8: Contribution Snake

Set up the snake action in `.github/workflows/snake.yml`:

```yaml
name: Generate Snake
on:
  schedule: [{ cron: "0 0 * * *" }]
  workflow_dispatch:
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: Platane/snk@v3
        with:
          github_user_name: yourusername
          outputs: |
            dist/github-contribution-grid-snake.svg
            dist/github-contribution-grid-snake-dark.svg?palette=github-dark
      - uses: crazy-max/ghaction-github-pages@v4
        with:
          target_branch: output
          build_dir: dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## 💡 Pro Tips

1. **Collapsible sections keep it clean**: The `<details>` tags collapse everything by default so visitors see the header/wave/greeting first. They expand what interests them.

2. **Auto-update everything**: Set up GitHub Actions for blog posts (blog-post-workflow), WakaTime (waka-readme), recent activity (recent-activity), and snake (Platane/snk). All run on cron.

3. **Trophy column count**: `column=8` fits most screens. Reduce to `column=4` on mobile or use `&column=-1` for auto-fit.

4. **Wave animation color**: `color=timeGradient` gives a gradient that changes over time. Replace with static `color=ff0000,00ff00,0000ff` for a fixed rainbow.

5. **Stats card order**: Put GitHub API stats first, streak second, WakaTime third. This gives the best visual flow with varying widths.
